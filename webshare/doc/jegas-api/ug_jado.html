ODBC itself uses a series of functions to do different kinds of queries to the connected database. Unfortunately; each has it's own "way" it needs to be used... So - in an effort to make the Jegas, LLC JADO Connector work identical for both ODBC, MYSQL, etc... (Speaking about how the JADO_RECORDSET class works) We've got these constants that guide requests through the proper code for their respective ODBC API command to be carried out.
