Prepared Variables for a New Firing of the thread. Note: You want to override this. Note its called AFTER each Execute my thread, AND in the constructor. This is to make it so if thhe thread has a lot of initialization, its "ready to go" when you fire it off, does its task, then setups for the next go around. Seems more efficient then waiting for init code at the start of each "FIRING" of the thread.
